Dynamic plugin glossary
  • Activated: indicate the state of a plugin that is under use. Note that in the 2.1 model a plugin is only activated when one of its class is loaded;
  • Deactivated: indicate the state of a plugin whose class have not been loaded yet;
  • Deactivation: process consisting in changing a plugin state from activated to deactivated;
  • Enabled: indicate the state of a plugin relatively to a given configuration. When a plugin is enabled its functionnalities are available. In 2.1, a plugin is enabled at startup if all its prerequisites are available (for more details see the current model page);
  • Disabled: indicate the state of a plugin relatively to a given configuration. A plugin is disabled if its prerequisites are not satisfied. In 2.1 a plugin can only be disabled at startup time.
  • Disablement: action of changing a plugin state from enable to disable at runtime. The disablement can only be done is the plugin is deactivated.
